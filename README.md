During guided deployment, use the following:
Stack Name: bayan-prod
Region: us-east-1
Allow SAM CLI IAM role creation: Y
Disable rollback: N

Once deployment is complete, the terminal will output two essential URLs:

WebSocketURL (e.g., wss://xxxx.execute-api.us-east-1.amazonaws.com/prod)

BaseUrl (e.g., https://yyyy.execute-api.us-east-1.amazonaws.com/Prod/)
 Frontend Setup (Local Development)
Link the frontend to your newly deployed AWS backend.

Bash
# Navigate to the frontend directory
cd bayan-frontend

# Install dependencies
npm install
Configure Environment Variables:
Create a .env file in the root of your bayan-frontend folder and add the URLs generated by your SAM deployment:

Code snippet
VITE_API_URL=[https://yyyy.execute-api.us-east-1.amazonaws.com/Prod](https://yyyy.execute-api.us-east-1.amazonaws.com/Prod)
VITE_WEBSOCKET_URL=wss://[xxxx.execute-api.us-east-1.amazonaws.com/prod](https://xxxx.execute-api.us-east-1.amazonaws.com/prod)
(Note: Ensure there are no trailing slashes at the end of the WebSocket URL)

Run the app:

Bash
npm run dev
Open http://localhost:5173 in your browser.

🛠️ Troubleshooting
Issue: Progress bar gets stuck on "Initializing AI Pipeline..."

Cause: The S3 bucket failed to trigger the Step Function, OR the Step Function lacks IAM permissions.

Fix: Check AWS Step Functions Executions in the console. Ensure the template.yaml uses !Ref for Lambda policies instead of !GetAtt Function.Arn.

Issue: SubscriptionRequiredException in Transcribe/Polly Lambdas.

Cause: The AWS account has a 24-hour verification hold on ML services, or is an educational account.

Fix: Ensure your AWS account has a valid credit card attached and has passed the 24-hour initial anti-fraud hold. For immediate academic demos, utilize the "Mock Bypass" by returning hardcoded JSON in the TranscribeLauncherFunction.

Issue: WebSocket connects, disconnects (Code 1006), then connects again.

Cause: React 18 Strict Mode mounts/unmounts useEffect twice in development.

Fix: This is normal development behavior. As long as the final console log says "Connected Successfully," the pipeline will work.
